# Инструкция по развертыванию трекера в интернет

## Вариант 1: Firebase + GitHub Pages (Рекомендуется)

### Шаг 1: Настройка Firebase

1. Перейдите на https://console.firebase.google.com/
2. Создайте новый проект (или используйте существующий)
3. В настройках проекта найдите "Your apps" → добавьте Web-приложение
4. Скопируйте конфигурацию Firebase
5. Откройте файл `firebase-config.js` и замените значения:
   ```javascript
   const firebaseConfig = {
       apiKey: "ваш_api_key",
       authDomain: "ваш_project_id.firebaseapp.com",
       // ... и т.д.
   };
   ```

6. В Firebase Console перейдите в "Realtime Database"
7. Создайте базу данных (выберите режим "Test mode" для начала)
8. Скопируйте URL базы данных в `firebase-config.js` в поле `databaseURL`

### Шаг 2: Настройка правил безопасности Firebase

В Firebase Console → Realtime Database → Rules, установите:

```json
{
  "rules": {
    "refactoringTracker": {
      ".read": true,
      ".write": true
    }
  }
}
```

**Внимание:** Эти правила позволяют всем читать и писать. Для продакшена настройте авторизацию!

### Шаг 3: Развертывание на GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите все файлы проекта:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/ваш-username/ваш-репозиторий.git
   git push -u origin main
   ```

3. В настройках репозитория → Pages:
   - Source: выберите branch `main`
   - Folder: `/ (root)`
   - Сохраните

4. Ваш трекер будет доступен по адресу:
   `https://ваш-username.github.io/ваш-репозиторий/`

### Шаг 4: Настройка домена (опционально)

Можно подключить свой домен в настройках GitHub Pages.

## Вариант 2: Netlify (Проще, но требует Firebase)

1. Зарегистрируйтесь на https://www.netlify.com/
2. Подключите ваш GitHub репозиторий
3. Netlify автоматически развернет сайт
4. Настройте Firebase как в Варианте 1

---

## Вариант 3: Vercel

1. Зарегистрируйтесь на https://vercel.com/
2. Импортируйте GitHub репозиторий
3. Автоматическое развертывание
4. Настройте Firebase

---

## Безопасность

⚠️ **Важно:** Текущая конфигурация Firebase позволяет всем читать и писать данные.

Для продакшена рекомендуется:
1. Включить Firebase Authentication
2. Настроить правила безопасности в Realtime Database
3. Ограничить доступ только авторизованным пользователям

---

## Альтернатива: Использование Google Sheets API

Если не хотите настраивать Firebase, можно использовать Google Sheets как базу данных. Это требует дополнительной настройки, но проще для не-разработчиков.

---

## Проверка работы

После развертывания:
1. Откройте сайт в браузере
2. Откройте в другом браузере/устройстве
3. Внесите изменения в одном окне
4. Изменения должны появиться во втором окне автоматически (через несколько секунд)

---

## Поддержка

Если возникли проблемы:
1. Проверьте консоль браузера (F12) на ошибки
2. Убедитесь, что Firebase конфигурация правильная
3. Проверьте правила безопасности в Firebase Console
